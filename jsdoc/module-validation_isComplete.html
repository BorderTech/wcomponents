Module: validation/isComplete

validation/isComplete

Provides functionality to test if a component is "complete". What constitutes complete is dependent upon the WComponent, but for most of them we have a simple test using an extension of module:wc/dom/isSuccessfulElement.

Requires

  • module:wc/Observer
  • module:wc/array/toArray
  • module:wc/dom/getFilteredGroup
  • module:wc/dom/isSuccessfulElement
  • module:wc/dom/tag
  • module:wc/dom/Widget
  • module:validation/validationManager

Methods

<static> isComplete(element) → {boolean}

Determines if an element is 'complete' i.e. it send something other than an empty string to the server. This public method is a fit for most components. Only unusual or complex components would need to have specialised methods for determining completeness.

Parameters:
Name Type Description
element Element

the element to test.
Returns:

true if the element is complete.

Type
boolean

<static> isCompleteHelper(container, widget, filter, theOtherThis) → {boolean}

A function to determine if a particular complex component is complete. Most WAI-ARIA based and compound controls can use this helper to determine their completeness just by passing in the Widget which describes their top level component and a filter/completeness test function.

Parameters:
Name Type Argument Description
container Element

A DOM node, usually one containing components but could be the component.
widget module:wc/dom/Widget

A Widget describing the component calling this function.
filter function

A function which returns true if an instance of the component is complete.
theOtherThis object <optional>

A reference to a "this" to pass to Array.some if the filter func needs this.
Returns:

true if complete. Note: we assume false because a component cannot be complete if the container does not contain any of them.

Type
boolean

<static> isContainerComplete(container) → {boolean}

Tests is a container is complete. A container is complete if ANY of the components it contains is complete, not if all of the components it contains is complete. Determining if an element is complete is done in two parts:

  1. we do observer.notify for any subscribers so that WAI-ARIA role based widgets can do their thing;
  2. if result is still false after 1 we do a DOM based test of likely candidates.
Parameters:
Name Type Description
container Element

That which we are testing.
Returns:

true if the container is "complete".

Type
boolean

<static> subscribe(subscriber) → {function}

Allow components to subscribe to this module so that they can look after their own notions of completeness.

Parameters:
Name Type Description
subscriber function

The function that will be notified by analytics/validationManager. This function MUST be present at "publish" time, but need not be preset at "subscribe" time (i.e. when this function is called).
See:
  • module:wc/Observer#subscribe
Returns:

subscriber A subscriber to ValidationManager, actually the subscriber argument function.
