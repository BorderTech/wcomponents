Element ui:fileUpload
Namespace https://github.com/bordertech/wcomponents/namespace/ui/v1.0
Annotations

WFileWidget represents component which can be used to upload zero or one file; WMultiFieldWidget represents a component which is used to upload zero or more files. The component is indicated by the @maxFiles attribute.

WMultiFileWidget expect that the POSTed form data contains:

Field name Type Mandatory Value
@id + ".selected" List yes (if maxFiles != 1) otherwise forbidden list of file indices

Attributes
QName Type Use Annotation
acceptedMimeTypes xs:string optional
A list of MIME types which may be uploaded using the fileupload component. If not set then no type constraints will be applied to the uploaded file.
accessibleText xs:string optional
Deprecated. Use toolTip instead.
ajax xs:boolean optional
Flag if the uploaded files will be AJAX triggers.
async xs:boolean optional
Flag used to allow us to distinguish between the multi and single file uploader. Default is true.
cols xs:int optional
Number of cols to use when displaying the uploaded files. If not set then the theme default will apply.
disabled xs:boolean optional
The component will be in a disabled state if "true". Not output if "false".
dropzone xs:string optional
The ID of an element that will receive "drag and dropped" files for this file uploader.
hidden xs:boolean optional
The component will be in a hidden state if "true". Not output if "false".
id xs:ID required
The unique identifier for the component.
maxFileSize xs:positiveInteger optional
The maximum file size (in bytes) which may be uploaded using the file upload. If this is not set then no explicit constraint is set but implicit constraints may still exist.
maxFiles xs:positiveInteger optional
The maximum number of files which may be uploaded using the file upload. If this is exactly 1 then the component is a simple single file upload. If this is not set then no explicit limit is set on the number of files which may be uploaded using the file upload.
readOnly xs:boolean optional
The component will be in a non-interactive state if "true". Not output if "false".
required xs:boolean optional
The component will be mandatory if "true". Not output if "false".
tabIndex xs:int optional
Sets the component's position in the tab navigation order. Not implemented by default as it may pose significant accessibility and usability problems.
toolTip xs:string optional
Text to display to the user as optional 'helper' information. Note, however, that there are some accessibility and usability implications of this attribute and it should be used with care.
track xs:boolean optional
Indicates that the component should take part in web analytics tracking.
