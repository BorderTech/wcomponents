Element ui:dialog
Namespace https://github.com/bordertech/wcomponents/namespace/ui/v1.0
Annotations
WDialog represents a hideable container for content. When open the container is bound to the underlying application but may provide modal behaviour as determined by the @modal attribute.

QName Type Default Use Annotation
accessKey restriction of xs:string optional
Sets a single modified key access key for a control. This attribute should be used with caution as:
  • some user agents reserve some keys for internal use and do not expose them to the client for use as access keys; and
  • some keys may clash or cause issues with use of various assistive technologies.
accessibleText xs:string optional
Deprecated. Use toolTip instead.
cancel xs:boolean false optional
If true then the component acts as a cancel control and the user will be warned of unsaved changes.
class xs:string optional
Provides the ability to pass an HTML class attribute onto the root output element.
disabled xs:boolean false optional
The component will be in a disabled state if "true".
hidden xs:boolean false optional
The component will be in a hidden state if "true".
id xs:ID required
The unique identifier for the component.
imagePosition restriction of xs:NMTOKEN optional
The position of the image specified in the imageUrl attribute relative to any text content in the control. If imageUrl is not set then this attribute is ignored.
imageUrl xs:anyURI optional
The location of an image to display in the control.
msg xs:string optional
If set then the component will require confirmation before submission. This can be used to add a custom unsaved changes warning if @cancel is "true".
popup xs:boolean optional
Indicates whether this button will trigger a WPopup. Required for WCAG2.0 level A compliance of WPopup.
tabIndex xs:int optional
Sets the component's position in the tab navigation order. Not implemented by default as it may pose significant accessibility and usability problems. Under review: may be removed.
toolTip xs:string optional
Text to display to the user as optional 'helper' information. Note, however, that there are some accessibility and usability implications of this attribute and it should be used with care.
track xs:boolean false optional
Indicates that the component should take part in web analytics tracking.
type restriction of xs:NMTOKEN optional
Indicates the rendered appearance of the component. The default is the native output of the control (button for buttons, link for links).
unsavedChanges xs:boolean optional
If set and @cancel is true the user will receive a warning even if they have not made changes on the current screen.
validates xs:NMTOKEN optional
Indicates the region of the view which will be validated by the control's validating action.
