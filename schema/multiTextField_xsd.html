WMultiTextField represents a compound multi-input device for simple text input.

WMultiTextField expects that the POSTed form data contains:

Field name Type Mandatory Value
@id String yes a list of text values entered in the fields

The maximum number of inputs which are to be made available to the user.

The maximum number of characters which constitute valid input if the user makes any input at all . If not set then no maximum is set but one may be implied by the user agent.
minLength xs:positiveInteger optional
The minimum number of characters which constitute valid input if the user makes any input at all . If not set then no minimum is required.
pattern xs:string optional
A HTML5 regular expression pattern which applies to the content of the input as a whole.

Used to provide an indication of the type of input expected. Not a replacement for a label or title.

The component will be in a non-interactive state if "true".

Sets the component's position in the tab navigation order. Not implemented by default as it may pose significant accessibility and usability problems.

Text to display to the user as optional 'helper' information. Note, however, that there are some accessibility and usability implications of this attribute and it should be used with care.
