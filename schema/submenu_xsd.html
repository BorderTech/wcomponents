Element ui:submenu
Namespace https://github.com/bordertech/wcomponents/namespace/ui/v1.0
Annotations

WSubMenu provides a grouping mechanism for a set of items with in a WMenu where those items have an intrinsic sematic group nature. In practice a submenu is a hideable set of items which is indicated by a label and interacting with the label toggles the hidden state of the submenu's content.

WSubMenu expects that the POSTed form data contains:

Field name Type Mandatory Value Notes
@id String no "x" Value POSTed only if the WSubMenu is the item causing the form submission (using mode server or as AJAX).
@id.open Boolean no "true" if open, null or "false" if closed.
@id.selected String no "x" Value POSTed only if the WSubMenu is in the selected state.

Note: WSubMenu expect its parent Menu to put the "-h" parameter in the request to indicate the sub-menu and its items have been included in the request.

selectMode restriction of xs:NMTOKEN optional
Indicates that the items in the menu may be "selected". If not present then "none" is implied. Selection is additive with navigation and command invocation.
selectable xs:boolean optional
Indicates that the sub menu may be selected as an entity separate from its contained components. Explicitly output if explicitly set "false" to override a parent component's selectMode.
selected xs:boolean optional
