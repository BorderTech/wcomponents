Namespace https://github.com/bordertech/wcomponents/namespace/ui/v1.0
Annotations

WDropdown represents a single selection device which does not allow for no selection.

WDropdown expects that the POSTed form data contains:

Field name Type Mandatory Value Notes
@id String no the value of the selectied option Note that whilst the value is not mandatory a value will always be POSTed for a WDropdown unless the control has been externally manipulated.

QName Type Default Use Annotation
accessibleText xs:string optional
Deprecated. Use toolTip instead.
autocomplete xs:string optional
This is the autocomplete setting for a component. If not set then the user agent can determine the autocomplete state. See https://html.spec.whatwg.org/multipage/forms.html#attr-fe-autocomplete
class xs:string optional
Provides the ability to pass an HTML class attribute onto the root output element.
data xs:string optional
The key name for the datalist. If set then only the selected option[s] is/are included in the source XML and all options and optgroups are fetched in a separate transaction.
disabled xs:boolean false optional
The component will be in a disabled state if "true".
hidden xs:boolean false optional
The component will be in a hidden state if "true".
id xs:ID required
The unique identifier for the component.
optionWidth xs:positiveInteger optional
Indicates the space allowed for the options in the dropdown if the type is combo. If not set then the options will occupy enough horizontal space to allow them to each occupy a single line of text.
readOnly xs:boolean false optional
The component will be in a non-interactive state if "true".
required xs:boolean false optional
The component will be mandatory if "true".
submitOnChange xs:boolean optional
This attribute has significant negative accessibility implications and extreme care should be taken with its implementation. It may be removed completely from future versions of WComponents because of these accessibility issues.
tabIndex xs:int optional
Sets the component's position in the tab navigation order. Not implemented by default as it may pose significant accessibility and usability problems.
toolTip xs:string optional
Text to display to the user as optional 'helper' information. Note, however, that there are some accessibility and usability implications of this attribute and it should be used with care.
track xs:boolean false optional
Indicates that the component should take part in web analytics tracking.
type restriction of xs:NMTOKEN optional
If set then the dropdown will allow free text input as well as selection from an exiting option.
