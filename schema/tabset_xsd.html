Annotations

WTabSet represents a container for a series of tabs with their content and optional separators.

WTabSet expects that the POSTed form data contains:

Field name Type Mandatory Value
@id String yes Array of open/selected tabs indices (zero-based), or "" for no open tabs
@id + ".showHeadOnly" String no "true" | "false"

Attributes
QName Type Default Use Annotation
class xs:string optional
Provides the ability to pass an HTML class attribute onto the root output element.
contentHeight xs:token optional
Indicates the height of the content box for the tab content for all tabs in this tabset. If any tab's content exceeds this height it will provide a scroll mechanism for that tab's content. If not set then th eheight of the tab content is determined by the user agent so as to display all content without scrolling.
disabled xs:boolean false optional
The component will be in a disabled state if "true".
groupName xs:NMTOKEN optional
Indicates that the collapsible is part of a group of collapsibles the open state of which is controlled by a WCollapsibleToggle.
hidden xs:boolean false optional
The component will be in a hidden state if "true".
id xs:ID required
The unique identifier for the component.
showHeadOnly xs:boolean optional
Deprecated: do not use.
single xs:boolean optional
Indicates an accordion should only open one tab at a time.
track xs:boolean false optional
Indicates that the component should take part in web analytics tracking.
type restriction of xs:NMTOKEN required
Indicates the layout and interaction type of the tab set. The actual interpretation of this attribute is theme dependant.
