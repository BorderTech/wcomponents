Attributes
QName Type Use Annotation
accessKey restriction of xs:string optional
Sets a single modified key access key for a control. This attribute should be used with caution as:
  • some user agents reserve some keys for internal use and do not expose them to the client for use as access keys; and
  • some keys may clash or cause issues with use of various assistive technologies.
cancel xs:boolean optional
If true then the component acts as a cancel control and the user will be warned of unsaved changes.
class xs:string optional
Provides the ability to pass an HTML class attribute onto the root output element.
disabled xs:boolean optional
The component will be in a disabled state if "true". Not output if "false".
hidden xs:boolean optional
The component will be in a hidden state if "true". Not output if "false".
id xs:ID required
The unique identifier for the component.
msg xs:string optional
If set then the component will require confirmation before submission. This can be used to add a custom unsaved changes warning if @cancel is true.
selectable xs:boolean optional
Indicates that the sub menu may be selected as an entity separate from its contained components. Explicitly output if explicitly set "false" to override a parent component's selectMode. Selectable defaults to the container. e.g. if the container has a selectMode of NONE, then selectable defaults to false.
selected xs:boolean optional
indicates that the submenu is selected. Not output if "false".
submit xs:boolean optional
Setting submit indicates that actioning this item should trigger a form submit. This attribute and @url are mutually exclusive.
targetWindow xs:NMTOKEN optional
Provides a machine-readable reference to a browser window. If set, and the @url attribute is set, then the link to the @url value will open in a new window with this as its window name. There are some implementation issues caused by long standing bugs in some common user agents and some keywords in the HTML specification.
toolTip xs:string optional
Text to display to the user as optional 'helper' information. Note, however, that there are some accessibility and usability implications of this attribute and it should be used with care.
track xs:boolean optional
Indicates that the component should take part in web analytics tracking.
url xs:anyURI optional
Setting a url indicates that actioning this item should trigger navigation. This attribute and @submit are mutually exclusive.
validates xs:NMTOKEN optional
Used in conjunction with client side validation (WComponents plugin since v14.0.001). If client side validation is enabled and "validates" not defined, then the entire form will be validated client side by any interaction unless the cancel property is set "true".
