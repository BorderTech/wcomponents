Element ui:collapsible
Namespace https://github.com/bordertech/wcomponents/namespace/ui/v1.0
Annotations

WCollapsible represents a container with content the visibility of which can be toggled in the client using a brief text label.

WCollapsible expects that the POSTed form data contains:

Field name Type Mandatory Value
@id String yes "open" or "closed"

Attributes
QName Type Use Annotation
class xs:string optional
Provides the ability to pass an HTML class attribute onto the root output element.
collapsed xs:boolean optional
Indicates that the collapsible should be in a closed state when it loads. Not output if "false".
groupName xs:NMTOKEN optional
Indicates that the collapsible is part of a group of collapsibles the open state of which is controlled by a WCollapsibleToggle.
hidden xs:boolean optional
The component will be in a hidden state if "true". Not output if "false".
id xs:ID required
The unique identifier for the component.
level restriction of xs:int optional
Indicates the heading level or priority. Level 1 has the highest priority and level 6 the lowest. There are strict requirements on heading use, hierarchy and nesting in the HTML specification and accessibility guidelines which have an impact on the use of headings.
mode restriction of xs:NMTOKEN required
Indicates the content display mode for the component
track xs:boolean optional
Indicates that the component should take part in web analytics tracking.
